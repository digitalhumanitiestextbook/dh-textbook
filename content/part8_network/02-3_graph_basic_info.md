## 2.3. 그래프 기본 정보 확인하기

앞선 2.2절에서 우리는 우리의 첫 번째 네트워크 그래프 `G`를 성공적으로 만들었습니다. 노드를 추가하고, 엣지를 연결하고, 각각에 의미 있는 속성까지 부여했습니다. 하지만 지금 우리의 그래프 `G`는 컴퓨터 메모리 안에만 존재하는, 눈에 보이지 않는 데이터 덩어리일 뿐입니다. 우리가 만든 이 작은 세계가 과연 어떤 모습인지, 의도대로 잘 만들어졌는지 확인하고 싶지 않으신가요?

이번 섹션에서는 우리가 생성한 그래프 객체(`G`)에게 "네 자신에 대해 말해봐!"라고 질문을 던지는 방법을 배웁니다. 즉, 그래프에 어떤 노드와 엣지들이 들어있는지, 그 개수는 몇 개인지, 각 노드와 엣지는 어떤 속성을 갖고 있는지 등을 확인하는 기본적인 명령어들을 익힐 것입니다. 이 과정은 앞으로 우리가 더 복잡한 분석을 수행하기에 앞서, 데이터가 올바르게 구성되었는지 확인하고 네트워크의 전반적인 규모를 파악하는 필수적인 '탐색적 데이터 분석(Exploratory Data Analysis)'의 첫걸음입니다.

**※ 실습을 위한 준비 (Context Setting)**

이번 2.3절의 모든 코드는 바로 앞 2.2절에서 우리가 함께 만들었던 가상의 '프렌즈' 그래프 `G`를 그대로 사용합니다. 혹시 Colab 노트북을 새로 시작했거나 이전 내용이 지워졌을 경우를 대비하여, 아래의 준비 코드를 먼저 실행하여 `G`를 다시 만들어주세요.

(아래 코드를 복사하여 Colab 코드 셀에 붙여넣고 실행하세요.)

```
import networkx as nx

# 1. 빈 그래프 객체 생성
G = nx.Graph()

# 2. 노드 추가 (개별, 리스트, 속성 부여)
G.add_node("Monica")
character_list = ["Rachel", "Ross", "Chandler", "Joey", "Phoebe"]
G.add_nodes_from(character_list)
G.nodes["Phoebe"]["occupation"] = "Musician"
G.nodes["Phoebe"]["gender"] = "Female"

# 3. 엣지 추가 (개별, 리스트, 속성/가중치 부여)
G.add_edge("Monica", "Rachel")
edge_list = [("Monica", "Phoebe"), ("Chandler", "Joey")]
G.add_edges_from(edge_list)
G.add_edge("Monica", "Chandler", relationship="Spouse", weight=10)
G.edges[("Monica", "Rachel")]["relationship"] = "Childhood Friend"
```


---

### 2.3.1. 노드 관련 정보: G.nodes(), G.number_of_nodes(), G.nodes(data=True)

가장 먼저 우리 그래프에 어떤 '행위자'들이 있는지, 즉 노드 정보를 확인해 보겠습니다.

* **모든 노드 목록 보기: `G.nodes()`**

    `G.nodes()`는 현재 그래프 `G`에 포함된 모든 노드의 목록을 보여줍니다. 마치 파티에 참석한 사람들의 명단을 확인하는 것과 같습니다.

    아래 코드를 실행해 보세요.

```
    G.nodes()
```

실행하면 `NodeView(['Monica', 'Rachel', 'Ross', 'Chandler', 'Joey', 'Phoebe'])` 와 유사한 결과가 출력될 것입니다. 이는 `G` 안에 6명의 주인공 이름이 노드로 잘 들어가 있음을 의미합니다.

* **노드 개수 확인하기: `G.number_of_nodes()`**

    네트워크의 규모를 파악하는 가장 기본적인 정보는 노드의 개수입니다. `G.number_of_nodes()` 함수는 그래프에 몇 개의 노드가 있는지를 정수(integer)로 반환합니다.

    아래 코드를 실행해 보세요.

```
    G.number_of_nodes()
```

우리는 6명의 노드를 추가했으므로, 결과값으로 `6`이 출력될 것입니다. 수백, 수천 개의 노드를 다룰 때, 이 함수는 네트워크의 전체 크기를 한눈에 파악하게 해줍니다.

* **노드 속성 정보 함께 보기: `G.nodes(data=True)`**

    단순히 노드 이름만 보는 것을 넘어, 각 노드가 어떤 속성(attribute)을 가지고 있는지 확인하고 싶을 때가 많습니다. 이럴 때는 `G.nodes()` 함수에 `data=True` 라는 옵션을 추가합니다.

    아래 코드를 실행해 보세요.

```
    G.nodes(data=True)
```

결과는 이전보다 훨씬 더 자세할 것입니다. `NodeDataView([('Monica', {}), ('Rachel', {}), ... ('Phoebe', {'occupation': 'Musician', 'gender': 'Female'})])` 와 같은 형태로 출력됩니다. 각 노드의 이름과 함께, 그 노드가 가진 속성 정보가 파이썬의 **딕셔너리(dictionary)** `{}` 형태로 함께 표시됩니다. 우리는 'Phoebe' 노드에만 속성을 추가했기 때문에, 다른 노드들의 속성은 비어있는 딕셔너리 `{}`로 나타나는 것을 확인할 수 있습니다.

---

### 2.3.2. 엣지 관련 정보: G.edges(), G.number_of_edges(), G.edges(data=True)

이제 노드들을 연결하는 '관계', 즉 엣지 정보를 확인해 보겠습니다.

* **모든 엣지 목록 보기: `G.edges()`**

    `G.edges()`는 그래프에 존재하는 모든 엣지의 목록을 보여줍니다. 각 엣지는 `(노드1, 노드2)` 형태의 튜플(tuple)로 표시됩니다.

    아래 코드를 실행해 보세요.

```
    G.edges()
```

 실행 결과는 `EdgeView([('Monica', 'Rachel'), ('Monica', 'Phoebe'), ('Monica', 'Chandler'), ('Chandler', 'Joey')])` 와 같을 것입니다. 우리가 `.add_edge()` 와 `.add_edges_from()` 으로 추가한 4개의 연결이 잘 생성되었음을 의미합니다.

* **엣지 개수 확인하기: `G.number_of_edges()`**

    노드 개수와 마찬가지로, 전체 엣지의 개수는 네트워크가 얼마나 '연결'되어 있는지를 보여주는 기본 정보입니다. `G.number_of_edges()` 함수를 사용합니다.

    아래 코드를 실행해 보세요.

```
    G.number_of_edges()
```

우리는 4개의 엣지를 추가했으므로, 결과는 `4`가 됩니다.

* **엣지 속성 정보 함께 보기: `G.edges(data=True)`**

    엣지에 부여한 관계의 종류(`relationship`)나 강도(`weight`)와 같은 속성 정보를 확인하려면, `G.edges()` 함수에도 `data=True` 옵션을 사용합니다.

    아래 코드를 실행해 보세요.

```
    G.edges(data=True)
```

결과는 `EdgeDataView([('Monica', 'Rachel', {'relationship': 'Childhood Friend'}), ('Monica', 'Phoebe', {}), ('Monica', 'Chandler', {'relationship': 'Spouse', 'weight': 10}), ...])` 와 같은 형태로, 각 엣지를 구성하는 두 노드와 함께 해당 엣지의 속성 딕셔너리를 보여줍니다. 이를 통해 우리는 '모니카'와 '챈들러'의 관계는 '부부'이며 그 강도가 '10'이라는 구체적인 정보까지 한눈에 확인할 수 있습니다.

### 2.3.3. 특정 노드의 이웃 및 연결 차수(Degree)

전체 네트워크 정보 외에도, 특정 노드를 중심으로 한 미시적인 정보도 매우 중요합니다.

* **특정 노드의 이웃 찾기: `G.neighbors('노드이름')`**

    한 노드에 직접적으로 연결된 다른 노드들을 그 노드의 **이웃(neighbors)**이라고 합니다. `G.neighbors()` 함수에 노드 이름을 입력하면, 그 노드의 모든 이웃 목록을 보여줍니다. '모니카'의 이웃은 누구인지 확인해 봅시다.

    아래 코드를 실행해 보세요.

```
    list(G.neighbors("Monica"))
```

'Monica'는 'Rachel', 'Phoebe', 'Chandler'와 연결되어 있으므로, 이 세 명의 이름이 리스트 형태로 출력될 것입니다. (참고: `G.neighbors()`의 결과는 그대로 출력하면 조금 다른 형태로 보이므로, `list()` 함수로 감싸주면 우리가 보기 편한 리스트 형태로 변환됩니다.)

* **연결 차수(Degree) 확인하기: `G.degree()`**

    **연결 차수(Degree)**는 네트워크 분석에서 가장 기본적이면서도 중요한 개념으로, **한 노드가 몇 개의 엣지에 연결되어 있는지**를 나타내는 숫자입니다. 즉, 한 노드가 얼마나 많은 이웃을 가졌는지를 의미하며, 그 노드의 '연결성' 또는 '인기도'를 측정하는 가장 간단한 지표입니다. `G.degree()`는 그래프에 있는 모든 노드의 연결 차수를 계산해 줍니다.

    아래 코드를 실행해 보세요.

```
    G.degree()
```

결과는 `DegreeView([('Monica', 3), ('Rachel', 1), ('Ross', 0), ...])` 와 같이 각 노드와 그 노드의 연결 차수가 쌍으로 묶인 목록으로 나타납니다. '모니카'는 3개의 엣지에 연결되어 차수가 3이고, '로스'는 아직 아무 엣지에도 연결되지 않아 차수가 0임을 알 수 있습니다. 특정 노드의 연결 차수만 보고 싶다면 `G.degree("Monica")`와 같이 사용할 수도 있습니다.

>
> **간단 프롬프트:**
> `어떤 그래프 G의 G.nodes() 결과가 ['A', 'B', 'C']이고, G.edges() 결과가 [('A', 'B'), ('A', 'C')] 라면, 이 그래프는 어떻게 생겼을까?`
>
> **상세 프롬프트:**
> `어떤 미지의 그래프 G에 대해 다음 4가지 정보를 알아냈어.
> 1. G.number_of_nodes()는 4야.
> 2. G.number_of_edges()는 4야.
> 3. G.nodes(data=True)의 결과, 'David'라는 노드는 {'role': 'manager'} 속성을 가져.
> 4. `G.degree()`의 결과, ('David', 3) 이라는 값이 포함돼 있어.
> 이 정보들을 종합해서 이 미지의 그래프 G가 어떤 모습일지, 그리고 'David'가 이 네트워크에서 어떤 역할을 할지 인문학적 상상력을 발휘해서 이야기를 만들어 설명해줘.`
>
> **인문학적 프롬프트:**
> `'연결 차수(degree)'는 한 노드의 '중요도'를 측정하는 가장 단순한 지표야. 하지만 이게 항상 진정한 '영향력'을 의미할까? 예를 들어, 친구는 100명이지만 모두 각자 노는 사람과, 친구는 단 2명이지만 그 두 명이 각각 거대한 두 집단의 리더여서 그 두 집단을 연결하는 유일한 사람. 이 두 사람 중 누가 더 '중요'하거나 '영향력'있다고 말할 수 있을까? '연결 차수'라는 지표가 가진 명백한 유용성과, 그것만으로 파악할 수 없는 관계의 질적, 구조적 측면에 대해 논해줘.`

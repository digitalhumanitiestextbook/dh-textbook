## 6.4. "프렌즈" 관계망 시각화 실습 및 결과 해석

지금까지 우리는 네트워크를 분석하는 방법(4, 5장)과, 분석 결과를 시각적 요소로 표현하는 기술(6.3절)을 모두 배웠습니다. 이제 대장정의 마지막 단계는 이 모든 지식과 기술을 하나로 모아, "프렌즈"의 사회적 관계를 담은 최종적인 "지도"를 완성하고, 그 지도 속에서 우리가 미처 발견하지 못했던 새로운 이야기를 읽어내는 것입니다.

이번 섹션에서는 6.3절에서 배운 고급 시각화 기법을 총동원하여, 9명의 핵심 인물 네트워크(`G_core`)를 한 장의 정보가 풍부한 다이어그램으로 그리겠습니다. 그리고 완성된 그림을 함께 분석하며, 시각화가 어떻게 추상적인 데이터를 구체적인 인문학적 통찰로 바꾸어 놓는지 그 과정을 직접 체험해 보겠습니다.

### 1단계: 종합 시각화를 위한 최종 코드

우리의 목표는 다음과 같은 다층적인 정보를 하나의 시각화에 모두 담아내는 것입니다.

- **노드 크기:** 인물의 "총 상호작용량" (`가중 연결 중심성`)
- **노드 색상:** 인물의 "간접적 영향력" (`아이겐벡터 중심성`)
- **엣지 두께:** 두 인물 사이의 "관계의 강도" (`weight`)

아래 코드는 시각화에 필요한 데이터를 준비하는 과정부터 최종 그림을 그리는 것까지, 모든 단계를 포함하고 있습니다.

(앞선 코드에 이어서, 아래 코드를 다음 셀에 복사하여 실행하세요. `G_directed`와 `main_characters_100` 변수가 필요합니다.)

#### [코드]
```
# 필요한 라이브러리들을 불러옵니다.
import networkx as nx
import matplotlib.pyplot as plt

# --- 1. 시각화 대상 데이터 준비 ---
# (이전 단계에서 G_directed와 main_characters_100 변수가 생성되었다고 가정)
# 주요 인물 9명으로 구성된 하위 그래프를 생성합니다.
G_core = G_directed.subgraph(main_characters_100)

# --- 2. 시각화에 필요한 분석 값 계산 ---
# 가중 연결 중심성 (총 상호작용량)
total_strength = dict(G_core.degree(weight='weight'))
# 아이겐벡터 중심성 (간접적 영향력)
eigenvector_centrality = nx.eigenvector_centrality(G_core, weight='weight', max_iter=1000)

print("1단계: 시각화에 필요한 데이터 계산 완료")


# --- 3. 시각적 요소에 맵핑할 리스트 생성 ---
# 노드 크기 리스트 (총 상호작용량 기준, 20으로 나누어 스케일 조절)
node_sizes = [total_strength.get(node, 0) / 20 for node in G_core.nodes()]

# 노드 색상 리스트 (아이겐벡터 중심성 기준)
node_colors = [eigenvector_centrality.get(node, 0) for node in G_core.nodes()]

# 엣지 두께 리스트 (가중치 기준, 500으로 나누어 스케일 조절)
edge_widths = [d.get('weight', 0) / 500 for u, v, d in G_core.edges(data=True)]

print("2단계: 시각적 요소(크기, 색상, 두께) 리스트 생성 완료")


# --- 4. 모든 것을 통합한 최종 시각화 ---
plt.figure(figsize=(13, 13)) # 그림 크기를 넉넉하게
pos = nx.kamada_kawai_layout(G_core) # 미학적으로 균형 잡힌 레이아웃 선택

nx.draw(G_core, pos, with_labels=True, 
        node_size=node_sizes,          # 크기는 총 상호작용량
        node_color=node_colors,        # 색상은 아이겐벡터 중심성
        cmap=plt.cm.YlOrRd,          # 노란색(낮음) -> 붉은색(높음) 컬러맵
        width=edge_widths,             # 두께는 관계의 강도
        edge_color='lightgray',
        alpha=0.9,                     # 투명도 조절
        font_size=12,
        font_weight='bold')

plt.title("Friends Core Network: A Visual Summary of Relationships", fontsize=18)
plt.show()

print("3단계: 최종 종합 시각화 완료")
```

### 2단계: 시각화 결과 해석: 그림을 통해 새로운 이야기 발견하기

이제 완성된 그림을 함께 해석해 봅시다. 이 한 장의 다이어그램은 "프렌즈"의 사회 구조에 대해 어떤 새로운 이야기를 들려주고 있을까요?

- **한눈에 보이는 주인공들의 지배력:** 가장 먼저 눈에 띄는 것은 **주인공 6명(Rachel, Ross, Monica, Chandler, Joey, Phoebe)의 노드가 다른 3명(Janice, Mike, Frank)의 노드보다 월등히 크다**는 점입니다. 이는 5장에서 숫자로 확인했던 "총 상호작용량"의 차이가 시각적으로 명확하게 드러난 것입니다. 이 그림은 누가 이 드라마의 진짜 "주인공"인지를 굳이 설명하지 않아도 한눈에 알 수 있게 해줍니다.

- **권력의 중심, "모니카-챈들러" 커플:** 노드의 **색깔**은 "간접적 영향력"(아이겐벡터 중심성)을 나타냅니다. 가장 진한 붉은색을 띠는 노드는 **챈들러와 모니카**입니다. 이는 다른 모든 "중요한" 인물들과 가장 강하고 안정적인 관계를 맺고 있는 이들 커플이, 이 네트워크의 실질적인 "권력의 중심"임을 시각적으로 보여줍니다. 그들의 아파트가 드라마의 주 무대였던 것처럼, 그들의 관계 역시 전체 네트워크를 지탱하는 가장 핵심적인 기둥입니다.

- **핵심 로맨스의 시각적 증명:** 노드와 노드를 잇는 **선의 굵기**는 관계의 강도를 의미합니다. 그림에서 가장 굵은 선들은 단연 "**모니카-챈들러**"와 "**로스-레이첼**"을 연결하는 선들입니다. 이는 드라마의 핵심적인 두 로맨스 관계가, 다른 우정 관계보다 훨씬 더 빈번하고 강렬한 상호작용으로 이루어져 있음을 명확하게 보여줍니다. 숫자만으로는 파악하기 어려웠던 관계의 "무게"가 선의 굵기를 통해 직관적으로 전달됩니다.

- **구조가 서사를 말하다:** 이 그림은 "프렌즈"의 서사 구조 그 자체를 보여주는 한 편의 "지도"입니다. 우리는 이 지도를 통해, 주인공 6명이 끈끈한 핵심 그룹을 이루고(크고 가까운 노드들), 그 안에서 모니카와 챈들러가 안정적인 중심으로 자리 잡고 있으며(진한 색), 두 핵심 커플의 로맨스가 전체 관계의 가장 굵은 기둥을 이루고 있다는 복합적인 이야기를, 단 한 장의 그림으로 이해하고 다른 사람에게 설득력 있게 전달할 수 있습니다. 이것이 바로 데이터 시각화의 진정한 힘입니다.

### 🤖 AI와 함께 탐색하기: 나만의 네트워크 지도 그리기

**학습 목표:** 자신이 가진 데이터를 바탕으로, 어떤 정보를 어떤 시각적 요소에 매핑하여 "나만의 이야기"를 전달하는 네트워크 지도를 만들지 구상하는 연습을 합니다.

**간단 프롬프트:**
`내가 가장 좋아하는 영화의 등장인물 관계를 시각화하고 싶어. 어떤 정보를 시각화하면 재미있을까?`

**상세 프롬프트:**
`내가 분석하고 싶은 대상은 J.R.R. 톨킨의 소설 "반지의 제왕" 속 "반지 원정대" 9명이야. 이들의 관계를 종합적으로 시각화하는 아이디어를 제안해줘.`
`1. 무엇을 기준으로 "노드 크기"를 정하면 좋을까?`
`2. 무엇을 기준으로 "노드 색상"을 정하면 좋을까? (예: 종족-호빗, 엘프, 인간 등)`
`3. 무엇을 기준으로 "엣지 두께"를 정하면 좋을까?`
`이 시각화를 통해 "반지 원정대"의 어떤 숨겨진 특징을 발견할 수 있을지 설명해줘.`

**인문학적 프롬프트:**
`우리가 만든 "프렌즈" 네트워크 시각화는 결국 우리의 "해석"이 담긴 한 편의 "주장"이야. 만약 우리가 노드 색상을 "성별"로, 노드 크기를 "매개 중심성"으로 정했다면, 이 그림은 완전히 다른 이야기를 했을 거야. 이처럼 시각화는 객관적인 진실의 반영이 아니라, 연구자가 데이터를 통해 자신의 주장을 펼치는 "수사학적 행위"라는 점에 대해 논해줘. 시각화 결과물을 접하는 독자로서, 우리는 그림 뒤에 숨겨진 연구자의 "선택"과 "의도"를 어떻게 비판적으로 읽어내야 할까?`

## 2.1. NetworkX 라이브러리 소개

우리가 네트워크라는 "건축물"을 짓기로 마음먹었다면, 가장 먼저 해야 할 일은 우리의 "공구함"을 준비하는 것입니다. 파이썬이라는 거대한 작업 공간에서, NetworkX는 네트워크의 생성, 분석, 시각화를 위해 특별히 설계된 최고의 전문 공구함, 즉 "**라이브러리(Library)**"입니다.

라이브러리란, 특정 종류의 작업을 쉽고 효율적으로 수행할 수 있도록 미리 만들어진 코드의 모음입니다. 우리가 직접 망치와 못을 만드는 대신 공구함에서 꺼내 쓰듯, NetworkX 라이브러리를 사용하면 복잡한 네트워크 알고리즘을 직접 구현할 필요 없이, 간단한 명령 몇 줄만으로 "노드 추가", "엣지 연결", "중심성 계산"과 같은 고수준의 작업을 수행할 수 있습니다.

이번 섹션에서는 이 강력한 공구함인 NetworkX가 무엇인지 알아보고, 실제 사용을 위해 설치하고 작업 공간으로 불러오는 방법을 배우겠습니다.

### 2.1.1. NetworkX란 무엇인가? (주요 기능 및 장점)

**NetworkX**는 파이썬 프로그래밍 언어를 위한 네트워크 분석 패키지입니다. 간단히 말해, 파이썬으로 복잡계 네트워크의 구조, 동학, 기능을 만들고, 조작하고, 연구하기 위해 만들어진 가장 대표적인 도구입니다. 인문학 연구자에게 NetworkX가 매력적인 이유는 다음과 같은 장점들 때문입니다.

* **직관적이고 쉬운 사용법:** NetworkX는 파이썬의 기본 자료구조(딕셔너리, 리스트 등)를 기반으로 설계되어, 프로그래밍 초심자도 비교적 쉽게 네트워크를 생성하고 데이터를 다룰 수 있습니다. 예를 들어, `G.add_node("Monica")` 와 같은 거의 자연어에 가까운 명령어로 노드를 추가할 수 있습니다.

* **다양한 네트워크 유형 지원:** 1장에서 살펴본 거의 모든 종류의 네트워크를 지원합니다. 친구 관계 같은 상호적 관계를 위한 **무방향 그래프(Graph)**, 영향력이나 인용 관계처럼 흐름이 있는 관계를 위한 **방향 그래프(DiGraph)**, 그리고 여러 종류의 관계가 중첩된 **다중 그래프(MultiGraph)** 까지 손쉽게 생성하고 분석할 수 있습니다.

* **풍부한 내장 알고리즘:** 네트워크 분석의 핵심은 다양한 분석 지표를 계산하는 것입니다. NetworkX에는 연결 중심성, 매개 중심성, 근접 중심성과 같은 주요 **중심성(Centrality)** 지표는 물론, 최단 경로, 클러스터링 계수, 커뮤니티 탐지 등 수많은 표준 네트워크 분석 알고리즘이 내장되어 있습니다. 우리는 이 알고리즘의 복잡한 수학적 원리를 모두 알지 못해도, `nx.degree_centrality(G)` 와 같은 간단한 함수 호출만으로 결과를 얻을 수 있습니다.

* **유연한 데이터 통합:** 파이썬 생태계의 가장 큰 장점은 다른 라이브러리와의 연동이 자유롭다는 것입니다. 우리는 데이터를 표 형태로 다루는 데 탁월한 **Pandas** 라이브러리로 "프렌즈" 대본 데이터를 정리하고, 그 결과를 NetworkX로 넘겨 네트워크를 구축한 뒤, 시각화 전문 라이브러리인 **Matplotlib**이나 **Seaborn**을 이용해 분석 결과를 아름다운 그래프로 표현할 수 있습니다. 이처럼 각 분야 최고의 도구들을 유기적으로 연결하여 작업할 수 있는 환경을 제공합니다.

* **활발한 커뮤니티와 풍부한 문서:** NetworkX는 전 세계 수많은 연구자와 개발자가 함께 만들어가는 오픈소스 프로젝트입니다. 따라서 기능이 계속해서 개선되고 있으며, 사용법을 상세히 설명하는 공식 문서(documentation)와 궁금한 점을 질문하고 답을 얻을 수 있는 온라인 커뮤니티가 매우 활성화되어 있습니다.

이러한 장점들 덕분에 NetworkX는 사회과학, 컴퓨터과학, 생물학은 물론 디지털 인문학에 이르기까지, 네트워크를 연구하는 거의 모든 분야에서 표준적인 도구로 자리 잡았습니다.

### 2.1.2. 설치 (!pip install networkx)

우리의 Colab 노트북에서 NetworkX라는 전문 공구함을 사용하려면, 먼저 이 공구함을 작업장으로 "가져오는" 과정, 즉 "**설치(Installation)**"가 필요합니다. 다행히 NetworkX는 매우 유명한 라이브러리이기 때문에, 대부분의 최신 Colab 환경에는 이미 설치가 되어 있습니다. 하지만, 설치가 되어있지 않거나 특정 버전을 사용해야 할 경우를 대비해 설치 방법을 알아두는 것은 중요합니다.

Colab에서 파이썬 라이브러리를 설치하는 명령어는 매우 간단합니다. 코드 셀에 아래 코드를 입력하고 실행(Ctrl+Enter 또는 Shift+Enter)하면 됩니다.

```
!pip install networkx
```

* `!` **(느낌표):** 이 기호는 Colab에게 "이것은 파이썬 코드가 아니라, 운영체제(리눅스)에 직접 내리는 명령어입니다"라고 알려주는 역할을 합니다.
* `pip` **:** 파이썬의 패키지 관리자로, 필요한 라이브러리를 인터넷 저장소(PyPI)에서 찾아 다운로드하고 설치해주는 프로그램의 이름입니다. "Pip Installs Packages"의 약자입니다.
* `install networkx` **:** "networkx" 라는 이름의 패키지를 설치하라는 명령어입니다.

이 명령어를 실행하면, 화면에 해당 라이브러리를 다운로드하고 설치하는 과정이 텍스트로 나타납니다. "Successfully installed networkx-..." 와 같은 메시지가 보인다면 성공적으로 설치가 완료된 것입니다. 이미 설치되어 있다면 "Requirement already satisfied" 라는 메시지가 뜰 것입니다.

### 2.1.3. 불러오기 (import networkx as nx)

라이브러리를 설치하는 것은 우리 컴퓨터에 특정 프로그램을 한 번만 설치하면 되는 것과 같습니다. 하지만 실제로 그 프로그램을 사용하려면, 매번 바탕화면의 아이콘을 더블클릭해서 실행해야 하죠. 파이썬 스크립트(혹은 Colab 노트북)에서 라이브러리를 사용하는 것도 마찬가지입니다. 설치된 라이브러리의 기능을 현재 작업에서 사용하겠다고 선언하는 과정이 필요한데, 이를 "**불러오기(import)**"라고 합니다.

NetworkX를 불러오는 명령어는 파이썬 세계의 오랜 관습처럼 굳어진 형태가 있습니다.

```
import networkx as nx
```

이 한 줄의 코드는 두 부분으로 나뉩니다.

* `import networkx` **:** "설치된 라이브러리 중에서 "networkx"를 이번 작업 세션으로 가져와서 사용하겠습니다." 라는 선언입니다.
* `as nx` **:** "앞으로 코드에서 "networkx"라는 긴 전체 이름 대신, "nx"라는 "**별명(alias)**"으로 부르겠습니다." 라는 의미입니다.

왜 굳이 별명을 사용할까요? `networkx.draw(G)`, `networkx.degree_centrality(G)` 처럼 매번 전체 이름을 입력하는 것은 번거롭고 코드가 길어집니다. `nx.draw(G)`, `nx.degree_centrality(G)` 처럼 짧은 별명을 사용하는 것이 훨씬 효율적입니다. `nx`는 NetworkX의 전 세계적인 표준 별명이므로, 이 관례를 따르는 것은 다른 사람들과 코드를 공유하고 소통하는 데 매우 중요합니다.

이제 우리는 `nx`라는 이름으로 NetworkX의 모든 강력한 기능을 사용할 준비를 마쳤습니다.

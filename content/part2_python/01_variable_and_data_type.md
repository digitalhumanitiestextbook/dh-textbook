## 변수와 데이터 타입 배우기

컴퓨터는 본질적으로 데이터를 처리하는 기계입니다. 우리가 컴퓨터에게 의미 있는 작업을 시키려면, 먼저 다루어야 할 정보를 파이썬이 인식할 수 있는 형태로 정리해야 합니다.

예를 들어 "프렌즈 주인공은 6명"이라는 정보를 다룬다고 해 봅시다. 여기에는 "6"이라는 숫자와 "주인공", "프렌즈"와 같은 글자가 섞여 있습니다. 컴퓨터에게 이를 전달해 파이썬에서 다루고자 하면 숫자는 숫자로, 글자는 글자로 구분해서 저장해야 합니다. 정확히 구분하지 않으면 컴퓨터가 숫자를 글자로 오인할 수도 있기 때문입니다. 이 과정에서 핵심적인 역할을 하는 것이 바로 "**변수(Variable)**"와 "**데이터 타입(Data Type)**"입니다.

## 변수(Variable): 데이터에 이름표 붙이기
변수(Variable)는, 간단히 말해, 데이터에 붙이는 이름표입니다. 예를 들어 "프렌즈" 로스의 유명한 대사 “We were on a break!”를 컴퓨터에 저장한다고 생각해 봅시다. 이 대사를 그냥 저장하면 나중에 필요할 때 다시 찾기가 매우 번거롭습니다. 

그래서 우리는 이 데이터에 `ross_quote`와 같은 식별 가능한 이름표를 붙여 줍니다. 이렇게 하면 나중에 `ross_quote`라는 이름표를 통해 컴퓨터는 `We were on a break!`라는 데이터를 즉시 가져올 수 있습니다. 이는 디지털인문학 과제 파일을 "디지털인문학_과제.hwp"라는 이름으로 저장하는 것과 같습니다. 의미 있는 이름을 붙여 두면 나중에 더욱 쉽게 찾을 수 있습니다. 만약 과제 파일명이 "asdf.hwp"라면 시간이 지나 그 파일을 찾기 어려울 뿐 아니라, 파일의 내용물이 무엇이었는지는 파일을 직접 열어 봐야 알 것입니다. 

그런데 이해하기 쉬운 이름을 붙이는 것 외에도, 파이썬에는 변수 이름을 지을 때 몇 가지 규칙이 있습니다.

* 영문자, 숫자, 밑줄(`_`)만 사용할 수 있습니다.
* 숫자로 시작할 수 없습니다. 예를 들어 `1st_character`는 사용할 수 없습니다.
* 띄어쓰기는 사용할 수 없습니다. 대신 밑줄(`_`)을 사용합니다. 예를 들어 `main character` 대신 `main_character`를 사용해야 합니다.
* 대소문자를 구분합니다. 예를 들어 `myVariable`과 `myvariable`은 다른 변수입니다.
* 파이썬이 이미 특별한 의미로 사용하고 있는 단어, 즉 예약어는 변수 이름으로 사용할 수 없습니다. `if`, `for`, `class` 등의 예시를 들 수 있습니다. 

변수에 데이터를 저장하는 과정을 할당(Assignment)이라고 하며, 등호(`=`) 기호를 사용합니다.

```
ross_quote = "We were on a break!"
```

위와 같이 `변수이름 = 데이터` 형식으로 작성하면, 오른쪽의 데이터가 왼쪽의 변수 이름표에 연결됩니다. 상자의 비유를 들어 "변수이름"이라는 상자에 "데이터"를 넣는다고 설명하기도 합니다. 

```
💡 자주하는 실수 
파이썬에서 등호 = 는 "할당"을 의미합니다. "같다"는 의미가 아닙니다!
"같은지 비교"할 때는 == 를 사용하며, 연산자 부분에서 보다 자세히 살펴보도록 하겠습니다.
```

## 데이터 타입(Data Type): 데이터의 종류 구분하기
데이터 타입(Data Type)은 데이터의 종류를 의미합니다. "레이첼 그린"처럼 글자로 이루어진 데이터도 있고, "프렌즈"의 총 시즌 수인 10처럼 소수점 없는 숫자로 된 데이터도 있으며, 에피소드 평점 8.4처럼 소수점이 있는 숫자도 있습니다.

또한, 여러 데이터를 순서대로 묶어 놓은 "목록" 형태의 데이터나, "이름: 레이첼, 직업: 패션"처럼 짝을 이룬 정보를 저장하는 데이터도 있습니다. "맞다/틀리다"를 나타내는 데이터 종류도 중요합니다.

파이썬은 데이터 타입에 따라 다른 방식으로 데이터를 처리합니다. 

* 숫자 2+3은 5가 됩니다.
* 글자 “안녕” + “하세요”는 이어 붙여서 “안녕하세요”가 됩니다.

따라서 어떤 종류의 데이터인지 명확히 구분하는 것은 대단히 중요합니다. 파이썬은 다양한 데이터 타입을 가지고 있지만, 이 장에서는 가장 기본적이고 자주 사용되는 여섯 종류를 "프렌즈" 예시와 함께 알아보겠습니다.

### 글자 데이터, 문자열(String) 다루기
파이썬에서 글자, 단어, 문장과 같은 텍스트 데이터를 통칭하여 문자열(String)이라고 부릅니다. 문자열 데이터를 코드에 쓸 때는, 데이터의 시작과 끝을 명확히 구분하기 위해 큰따옴표(`"`) 또는 작은따옴표(`'`)로 감싸야 합니다.

```
character_name = "모니카 겔러"
famous_line = 'We were on a break!'
```

따옴표로 감싸지 않으면 파이썬은 그것을 변수 이름이나 명령어로 착각합니다.

```
name = Monica  # 오류! Monica라는 변수를 찾으려고 함
name = "Monica"  # 정확! "Monica"라는 글자를 저장
```

큰따옴표(")와 작은따옴표(') 둘 다 사용 가능하지만, 한 문자열에서는 같은 종류로 시작하고 끝내야 합니다. 글자 안에 따옴표가 들어갈 경우, 다른 종류를 사용하면 편리합니다.

```
sentence1 = "He said, 'We were on a break!'"
sentence2 = 'She replied, "That is not true!"'
```

이제 문자열 개념의 숙지를 위해 간단한 코드를 실행시켜 보겠습니다.

```
# '모니카 겔러'라는 문자열 데이터를 character_name_monica 변수에 저장합니다.
character_name_monica = "모니카 겔러"  # 또는 'Monica Geller'

# 변수에 저장된 값을 출력합니다. f-string을 사용하면 변수와 텍스트를 함께 출력하기 편리합니다.
print(f"선택된 캐릭터 이름: {character_name_monica}")

# 변수에 저장된 데이터의 종류(타입)를 확인하고 출력합니다.
print(f"이름 정보의 종류: {type(character_name_monica)}")
```

**실행 및 설명**

* 결과 확인
    * `선택된 캐릭터 이름: 모니카 겔러`
    * `이름 정보의 종류: <class 'str'>`
* `=` 기호는 오른쪽의 `"모니카 겔러"`라는 문자열 값을 왼쪽의 `character_name_monica`라는 변수에 할당(저장)하는 역할을 합니다.
* `type()` 함수는 괄호 안의 변수나 값이 어떤 데이터 타입인지 알려줍니다. `<class 'str'>`은 이 데이터가 파이썬에서 문자열(String) 타입으로 인식된다는 것을 의미합니다.

### 소수점 없는 숫자 다루기 (정수, Integer)

10, 24, 0, -5처럼 소수점 없이 딱 떨어지는 숫자를 정수(Integer)라고 부릅니다. 파이썬에서는 정수를 코드에 직접 쓸 때 따옴표를 사용하지 않고 숫자 값만 그대로 적습니다.

```
int1 = 10
int2 = -5
```

이제 정수 개념의 숙지를 위해 간단한 코드를 실행시켜 보겠습니다.

```python
# 총 시즌 수 10을 total_seasons 변수에 저장합니다. 따옴표 없이 숫자만 사용합니다.
total_seasons = 10

# 변수 값과 타입을 출력합니다.
print(f"프렌즈 총 시즌 수: {total_seasons}")
print(f"시즌 수 정보 종류: {type(total_seasons)}")
```

**실행 및 설명:**

* 결과 확인
    * `프렌즈 총 시즌 수: 10`
    * `시즌 수 정보 종류: <class 'int'>`
* 숫자 10에는 따옴표가 없습니다. 파이썬은 이를 정수(Integer)로 인식하며, `type()` 함수는 `<class 'int'>`를 반환합니다.
* **중요**: 숫자 10과 문자열 "10"은 컴퓨터에게 완전히 다른 데이터입니다. 숫자 10은 계산이 가능하지만, 문자열 "10"은 글자로 취급됩니다.

### 소수점 있는 숫자 다루기 (실수, Float)

3.14, 22.5, -0.8, 10.0처럼 소수점 이하 부분을 포함하는 숫자를 컴퓨터 과학에서는 "**실수(Floating-point number)**"라고 하며, 파이썬에서는 줄여서 **Float** 타입이라고 부릅니다. 정수와 마찬가지로 코드에 직접 쓸 때는 따옴표 없이 숫자 그대로 사용합니다.

```
float1 = 3.14
float2 = -0.8
```
이제 실수 개념의 숙지를 위해 간단한 코드를 실행시켜 보겠습니다.

```python
# 첫 에피소드 평점 8.4를 episode1_rating 변수에 저장합니다.
episode1_rating = 8.4

# 변수 값과 타입을 출력합니다.
print(f"에피소드 1 평점: {episode1_rating}")
print(f"평점 정보 종류: {type(episode1_rating)}")
```

**실행 및 설명:**

* 결과 확인: `에피소드 1 평점: 8.4`, `평점 정보 종류: <class 'float'>`
* 소수점이 있는 숫자 8.4는 실수(Float) 타입으로 인식되며, `type()` 함수는 `<class 'float'>`를 반환합니다.

### 여러 데이터를 순서대로 묶기 (리스트, List)

여러 개의 데이터를 하나의 묶음으로, 넣은 순서대로 관리하고 싶을 때 리스트(List)를 사용합니다. 리스트는 대괄호(`[]`) 안에 각 데이터 항목, 요소(element)을 넣고, 항목 사이를 쉼표(`,`)로 구분하여 만듭니다. 

리스트 안에는 문자열, 숫자, 리스트 등 다양한 종류의 데이터를 섞어서 넣을 수도 있고, 값들을 채워 넣기 위해 빈 리스트를 생성할 수도 있습니다.

```
list1 = [1, True, 3.14, "리스트", []]
list2 = []
```
이제 리스트 개념의 숙지를 위해 간단한 코드를 실행시켜 보겠습니다.

```
# 주요 등장인물 이름을 리스트로 만들어 main_cast 변수에 저장합니다.
main_cast = ["모니카", "레이첼", "피비", "조이", "챈들러", "로스"] # 또는 영문 이름

# 변수에 저장된 리스트 전체를 출력합니다.
print("주요 등장인물 목록:", main_cast)

# 변수에 저장된 데이터의 타입(리스트)을 확인하고 출력합니다.
print("등장인물 목록 정보 종류:", type(main_cast))
```

**실행 및 설명**

* 결과 확인: `주요 등장인물 목록: ['모니카', '레이첼', '피비', '조이', '챈들러', '로스']`, `등장인물 목록 정보 종류: <class 'list'>`
* 대괄호 `[]` 안에 여러 개의 문자열(`str`)이 쉼표로 구분되어 순서대로 들어있는 것을 볼 수 있습니다. 이것이 바로 리스트(List)이며, 타입은 `list`로 표시됩니다.
* 리스트의 특징은 "순서가 있다"는 점과, 안에 있는 값을 나중에 "바꿀 수 있다"(mutable)는 점입니다.

### 이름표 붙은 정보 묶기 (딕셔너리, Dictionary)

리스트는 순서대로 데이터를 나열하지만, 각 데이터가 무엇을 의미하는지 알려주지는 않습니다. 만약 "이름은 레이첼", "직업은 패션"처럼 **이름표(Key)와 값(Value)을 짝지어** 정보를 저장하고 싶다면 "**딕셔너리(Dictionary)**"를 사용합니다. 

딕셔너리는 **중괄호(`{}`)** 안에 `Key: Value` 쌍들을 넣고, 각 쌍 사이를 "**쉼표(`,`)**"로 구분하여 만듭니다. Key는 보통 문자열이나 숫자를 사용하며, Value에는 어떤 데이터 타입이든 올 수 있습니다.

```
# dict1: 이름, 나이, 직업 등 사람의 기본 정보를 저장합니다.
dict1 = {
    '이름': '레이첼',  # 문자열 Key, 문자열 Value
    '나이': 29,         # 문자열 Key, 숫자 Value
    '국적': '미국'
}
```

이제 딕셔너리 개념의 숙지를 위해 간단한 코드를 실행시켜 보겠습니다.

```
# 챈들러 정보를 Key:Value 쌍으로 딕셔너리에 저장합니다.
chandler_info = {
  "이름": "챈들러 빙", # 또는 "Chandler Bing"
  "직업": "데이터 처리 전문가 (불확실)", # 또는 "IT Procurement Manager (Transponster?)"
  "파트너": "모니카" # 또는 "Monica"
}

# 변수에 저장된 딕셔너리 전체를 출력합니다.
print("챈들러 정보:", chandler_info)

# 변수에 저장된 데이터의 타입(딕셔너리)을 확인하고 출력합니다.
print("정보 묶음 종류:", type(chandler_info))
```

**실행 및 설명**

* 결과 확인: `챈들러 정보: {'이름': '챈들러 빙', '직업': '데이터 처리 전문가 (불확실)', '파트너': '모니카'}`, `정보 묶음 종류: <class 'dict'>`
* 중괄호 `{}` 안에 `Key: Value` 쌍들이 들어있는 것을 볼 수 있습니다. 이것이 딕셔너리(Dictionary)이며, 타입은 `dict`로 표시됩니다.
* 딕셔너리는 리스트와 달리 **순서가 중요하지 않습니다.** 대신, 고유한 Key(이름표)를 통해 원하는 Value(값)에 직접 접근할 수 있습니다. (예: `chandler_info['직업']`은 "데이터 처리 전문가 (불확실)" 값을 가져옵니다.)


### "맞다/틀리다" 표현하기 (불리언, Boolean)

프로그래밍에서는 어떤 조건이 "맞다" 또는 "틀리다"를 표현해야 할 때가 많습니다. 이때 사용하는 것이 "**불리언(Boolean)**" 데이터 타입입니다. 불리언은 오직 두 가지 값, **`True`** (참, 맞다)와 **`False`** (거짓, 틀리다) 만을 가집니다. 파이썬에서는 첫 글자를 대문자로 써야 합니다.

이제 불리언 개념의 숙지를 위해 간단한 코드를 실행시켜 보겠습니다.

```
# 로스와 레이첼이 연인이었는지 여부를 불리언 값으로 저장합니다.
ross_rachel_were_couple = True

# 변수 값과 타입을 출력합니다.
print(f"로스와 레이첼은 연인이었나?: {ross_rachel_were_couple}")
print(f"참/거짓 정보 종류: {type(ross_rachel_were_couple)}")
```

**실행 및 설명**

* 결과 확인: `로스와 레이첼은 연인이었나?: True`, `참/거짓 정보 종류: <class 'bool'>`
* `True`는 불리언(Boolean) 타입의 값이며, 타입은 `bool`로 표시됩니다.
* 비교 연산자(예: `5 > 3`)의 결과도 불리언 값(`True` 또는 `False`)으로 나옵니다. 불리언은 나중에 배울 조건문에서 매우 중요하게 사용됩니다.


## 데이터 타입은 왜 필요할까?
이번 섹션에서는 변수가 데이터를 저장하는 이름표 역할을 한다는 것과, 파이썬의 기본적인 데이터 타입인 문자열(`str`), 정수(`int`), 실수(`float`), 리스트(`list`), 딕셔너리(`dict`), 불리언(`bool`)에 대해 "프렌즈" 예시를 통해 자세히 알아보았습니다. 각 데이터 타입의 특징과 표기법, 그리고 사용 목적을 이해하는 것이 중요합니다.

그렇다면 파이썬이 다른 프로그래밍 언어와 구별되는 세 가지 중요한 특징을 통해, 타입이 왜 중요한지 더 깊이 이해해 봅시다.

### 1. 파이썬은 동적 타입(Dynamically Typed) 언어다.

파이썬에서는 변수에 데이터를 할당할 때(예: `character_name_monica = "모니카 겔러"`), 데이터 타입(문자열)을 따로 명시하지 않아도 됩니다. 파이썬 인터프리터가 자동으로 데이터 타입을 파악하여 변수에 연결합니다.

즉, 변수(character_name_monica) 자체가 특정 데이터 타입(문자열)을 가지는 것이 아닌, 변수가 지시하는 "값"이 데이터 타입을 가집니다. 그래서 같은 변수에 언제든 다른 타입의 데이터를 다시 할당할 수 있습니다.

| 코드 | 결과 | 설명 |
| :--- | :--- | :--- |
| `data = 10` | `<class 'int'>` | **data**는 정수 **10**을 가리킵니다. |
| `data = "10"` | `<class 'str'>` | **data**는 문자열 **"10"**을 가리킵니다. |


### 2. 타입이 다르면 연산을 할 수 없다. (ft. 왜 "2" + 2는 오류가 날까?)

"프렌즈"의 등장 인물들이 각자 다른 성격을 가져 같은 상황에서도 다른 반응을 보이듯이, 데이터 역시 타입에 따른 각자의 고유한 성격을 가지고 있습니다.

| 데이터 타입 | 주요 성격(연산) | 예시 |
| :--- | :--- | :--- |
| **정수(int)** | 산술 연산 (덧셈, 뺄셈, 곱셈 등) | `2 + 2` → `4` |
| **문자열(str)** | 결합 연산 (이어 붙이기), 반복 연산 | `"안녕" + "하세요"` → `"안녕하세요"` |

파이썬은 두 데이터의 타입을 엄격하게 구분하여, 의미가 모호한 연산은 허용하지 않습니다.

```
print("2" + 2)  # 오류 발생!
```

위 코드는 파이썬에게 “문자열 '2'와 정수 2를 어떻게 연산해야 하는가?”라는 모호한 질문을 하는 것과 같습니다. 두 값을 이어 붙여서 "22"로 만들라는 것인지, 아니면 정수 덧셈을 해서 4로 만들라는 것인지 알 수 없기 때문에 타입 오류(TypeError)를 발생시킵니다.

이를 해결하려면 의도에 맞게, 데이터 타입을 일치시킬 필요가 있습니다.

* 문자열 결합을 원할 경우: int를 str로 변환 → `print("2" + str(2))` → `"22"`
* 정수 덧셈을 원할 경우: str을 int로 변환 → `print(int("2") + 2)` → `4`


### 3. 불변형(Immutable)과 가변형(Mutable): 메모리 속 데이터의 변화

파이썬의 데이터 타입 중 정수(int), 실수(float), 문자열(str) 등의 데이터 타입은 대표적인 "불변형"(Immutable) 객체입니다. 이름 그대로 "변하지 않는 형태"라는 뜻입니다. 
불변형 데이터는 일단 한번 생성되면 그 내용을 바꿀 수 없습니다.

| 코드 | 변수 할당 | 메모리 변화 |
| :--- | :--- | :--- |
| `A = 10` | **A**는 정수 **10**을 가리킴. | 메모리 속 정수 **10**을 **A**가 가리키게 됨. |
| `A = 20` | **A**는 정수 **20**을 가리키도록 바뀜. | **A**가 새로운 정수 **20**을 가리키도록 연결이 바뀜 (기존 **10**은 그대로 유지). |

`A = 20`이라는 코드는 변수 A가 가리키는 데이터를 10에서 20으로 바꾼 것처럼 보입니다. 하지만 실제 컴퓨터 메모리 내부에서는 기존의 10을 수정하는 것이 아니라, 새로운 공간에 20이라는 객체를 만들고 변수 A가 그 새로운 20을 가리키도록 "연결"을 바꾼 것입니다.

이러한 불변형의 특징 덕분에, 파이썬은 프로그램의 안정성을 높이고 데이터의 신뢰성을 보장할 수 있게 됩니다. 반면, 리스트(List)나 딕셔너리(Dictionary) 같은 가변형(Mutable) 데이터는 내용이 직접 수정될 수 있습니다.

가변형(Mutable) 데이터, 즉 리스트(List)와 딕셔너리(Dictionary)는 불변형 데이터와 다르게 내용이 직접 수정될 수 있는 "상자"와 같습니다. 불변형이 값을 바꿀 때마다 새 객체를 만들어 연결을 바꾸는 반면, 가변형은 기존 메모리 공간을 그대로 유지하면서 그 안에 담긴 내용만 직접 수정합니다.

| 데이터 타입 특징 | 특징 (메모리 변화) | 예시 (리스트) |
| :--- | :--- | :--- |
| **가변형 (Mutable)** | 기존 메모리 주소는 유지하고 **내용만 변경**됩니다. | 친구 목록에서 두 번째 친구의 이름을 바꾸면 **목록 자체가 업데이트**됩니다. |

"프렌즈" 주인공들의 이사 기록을 리스트로 시뮬레이션하며 이해해 보겠습니다. 먼저, "프렌즈"의 주인공 여섯 명이 사는 아파트를 다음과 같이 리스트로 나타낼 수 있습니다.

```
프렌즈아파트 = ["모니카&레이첼", "조이&챈들러"]
```

모니카가 챈들러와 결혼하게 되면서, 레이첼은 피비와 함께 살게 됩니다. 리스트의 인덱스(순서) 기능을 이용하여 다음과 같이 변경 사항을 반영할 수 있습니다.

```
프렌즈아파트[0] = "모니카&챈들러"  # 0번 인덱스의 내용을 직접 수정
프렌즈아파트.append("레이첼&피비")  # 목록에 새로운 항목 추가
print(프렌즈아파트)  # 출력: ['모니카&챈들러', '조이&챈들러', '레이첼&피비'] 
```

이처럼 리스트는 항목을 추가(`.append()`), 삭제(`.pop()`), 수정하는 작업이 모두 가능합니다. 이는 리스트를 복사할 때 주의해야 하는 이유이기도 합니다. 단순히 = 기호를 사용해 복사하면 두 변수가 같은 메모리 공간을 "참조"하게 되어, 한쪽을 수정하면 다른 쪽도 함께 바뀌게 됩니다.
